def router(state: AgentState) -> str:
    phase = state.get("current_phase", "intake")
    if phase in ("intake", "recommendation",
            "calculation", "review", "generation"):
        return phase
    return END

workflow.set_conditional_entry_point(router)

Key Design Decision

Every node returns END after executing. The graph pauses and waits for the next user message. Forward navigation is controlled by the frontend, not the LLM.

This gives the sales rep full control over when to advance phases.

State Management

AgentState TypedDict

The typed state backbone flowing through every node

class AgentState(TypedDict):
    # Conversation
    messages: Annotated[list[BaseMessage], add_messages]
    session_id: str
    current_phase: Literal["intake", "recommendation",
        "calculation", "review", "generation"]

    # Phase 01 -- Customer Persona
    persona: Optional[CustomerPersona]

    # Phase 02 -- Product Recommendation
    selected_products: Optional[list[GetacProduct]]
    competitor_product_names: Optional[dict[str, str]]
    competitive_advantages: Optional[dict[str, list]]

    # Phase 03 -- TVO Calculation
    tvo_results: Optional[dict[str, TVOCalculation]]
    fleet_size: Optional[int]

    # Phase 04 -- Review
    proposal_approved: bool

    # Phase 05 -- Generation
    pptx_path: Optional[str]

Message Accumulation

Annotated[list, add_messages] uses a LangGraph reducer—new messages are appended, not replaced. The full conversation history flows through every node.

Typed Per Phase

Each phase writes to its own state fields. No shared mutable state—the TypedDict enforces boundaries at the type level.

MemorySaver Checkpointer

Every state update is checkpointed. The graph can resume from any point, enabling phase back-navigation.

Recommendation Phase

Product Selection Extraction

How It Works

1. Only last 3 human messages sent to extraction LLM

2. Returns {"selected": ["product_id"]} or {"remove": ["id"]}

3. Tagged with "extraction" — hidden from SSE stream

4. Matched against products.json catalog (4 Getac products)

Why separate from conversation? Product detection must be precise—temp=0, no streaming. The conversational LLM can freely discuss products without risking false selections.

def search_or_scrape(query, competitor_name, k=4):
    result = search_knowledge(query, k)

    # Check if results are relevant
    if result != NO_RESULTS_MSG \
       and _results_mention_competitor(
           result, competitor_name):
        return result

    # Gap detected -> auto-scrape
    content = scrape_competitor_for_rag(
        competitor_name
    )
    if content:
        ingest_text(content, source=source)
        return search_knowledge(query, k)

Provider Abstraction

LLM Provider Factory

def get_llm(
    provider: str | None = None,
    streaming: bool = True,
    temperature: float = 0.3,
) -> BaseChatModel:
    provider = provider or settings.llm_provider

    if provider == "ollama":
        from langchain_ollama import ChatOllama
        return ChatOllama(model=settings.ollama_model,
            streaming=streaming, temperature=temperature)

    elif provider == "openrouter":
        from langchain_openai import ChatOpenAI
        return ChatOpenAI(model=settings.openrouter_model,
            base_url="https://openrouter.ai/api/v1",
            streaming=streaming, temperature=temperature)

    elif provider == "openai":
        return ChatOpenAI(model=settings.openai_model,
            streaming=streaming, temperature=temperature)

Provider Table

Single Env Var

LLM_PROVIDER=openrouter switches the entire system. All nodes call get_llm()—none know which provider is active. Returns LangChain BaseChatModel.

Per-Call Config

streaming and temperature are configurable per invocation—extraction calls use streaming=False, temp=0; conversational calls use streaming=True, temp=0.3.
